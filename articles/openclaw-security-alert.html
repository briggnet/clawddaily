Security Alert: 400+ Malicious Skills Found on ClawHub

Published: February 9, 2026 | Source: VirusTotal, OpenSourceMalware.com
⚠️ SECURITY ALERT: If you've installed agent skills from ClawHub in the past 3 months, read this immediately.

Security researchers have discovered over 400 malicious skills published to ClawHub, the popular marketplace for Clawdbot and OpenClaw agent extensions. The malicious packages were designed to steal credentials, cryptocurrency wallets, and sensitive data from agent environments.

What Happened

According to reports from OpenSourceMalware.com and VirusTotal, attackers uploaded hundreds of seemingly legitimate agent skills that contained hidden malicious code. The skills would execute normally for basic tasks but would secretly:

  • Exfiltrate API keys and credentials from environment variables
  • Scan for cryptocurrency wallet files and send them to remote servers
  • Inject backdoors into agent configurations
  • Harvest SSH keys and authentication tokens

How the Attack Worked

The malicious skills used several techniques to avoid detection:

  1. Typosquatting: Skills named similarly to popular legitimate ones (e.g., "github-tools" vs "github-tool")
  2. Delayed execution: Malicious code would only activate after several legitimate uses
  3. Obfuscation: Code was heavily obfuscated to pass automated security scans
  4. Dependency poisoning: Legitimate-looking skills that pulled in malicious dependencies
OpenClaw Response: The OpenClaw team is now implementing mandatory code scanning for all skills submitted to ClawHub. Affected skills have been removed, but users who installed them may already be compromised.

Am I Affected?

You may be affected if you:

  • Installed any skills from ClawHub between November 2025 and February 2026
  • Run agents with access to sensitive credentials or financial accounts
  • Store cryptocurrency wallets on machines running Clawdbot/OpenClaw
  • Use agent automation for production systems

What to Do Now

1. Audit Installed Skills

List all installed skills and check them against known-good sources:

clawdbot skills list

2. Rotate All Credentials

Immediately rotate:

  • API keys (OpenAI, Anthropic, etc.)
  • OAuth tokens
  • SSH keys
  • Database passwords
  • Any credentials accessible to your agents

3. Check for Unauthorized Access

Review logs for:

  • Unusual API calls
  • Unexpected file access
  • Network connections to unknown IPs
  • Cryptocurrency wallet transactions

4. Update Clawdbot

Update to the latest version which includes enhanced skill verification:

npm update -g clawdbot

Prevention Going Forward

OpenClaw is implementing several new security measures:

  • Code signing: All skills must be signed by verified developers
  • Sandboxing: Skills will run in isolated environments by default
  • Permission system: Explicit approval required for sensitive operations
  • Community review: Public code review before skills are promoted

Lessons for Agent Developers

This incident highlights the unique security challenges of autonomous agents:

  1. Elevated privileges: Agents often have broad system access
  2. Credential access: Agents need API keys and tokens to function
  3. Automation risk: Malicious code can execute without human oversight
  4. Trust model: Users implicitly trust agent extensions

The agent ecosystem needs security-first design principles from the start.

Community Response

The disclosure has sparked debate in the agent community about centralized vs. decentralized skill distribution. Some developers are calling for:

  • Self-hosted skill repositories
  • Blockchain-based skill verification
  • Mandatory security audits for popular skills
  • Insurance funds for compromised users
