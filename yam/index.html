YAM Engine

A knowledge-seeking agent that learns everything through experience. No LLM. Every fact earned.

YAM is the next step after Potato. Potato was an embodied memory-driven agent. YAM pushes toward a self-contained knowledge system where search is perception, every result becomes memory, and retrieval uses meaning, relevance, life history, and recency to determine what surfaces.

The vector store remembers what a memory means. The graph remembers how that memory lives.

What YAM Is Not

Not an LLM. No generative model at the knowledge layer. Every fact in the graph was put there by a real answer to a real question. Nothing is in the graph that was not explicitly learned.

Not a chatbot. YAM does not generate text, summarize, or infer. It retrieves, ranks, and presents structured knowledge. When it does not know something, it says so.

Not RAG. Retrieval-augmented generation still uses a generative model at the output layer. YAM uses no generative model anywhere. The knowledge comes from the graph. The form comes from the retrieval ranking. Nothing is fabricated.

Architecture

The Parliament of Mind

YAM runs on the SPUD engine (Self-developing Perceptual Understanding Drive). The architecture has four components:

ComponentRolePortSigma
SPROUTAnalytical engine — seeks structure, facts, logical connections8001(0.4, 0.2, 0.2)
ROOTCreative engine — novel connections, analogies, unexpected patterns8002(0.5, 0.6, 0.3)
VEGCautious engine — risks, edge cases, what could go wrong8003(-0.1, 0.3, 0.2)
PATCH/YAMThe reasoning mind — integrates, learns, remembers8000(0.3, 0.3, 0.2)

Each engine has its own knowledge graph, its own emotional geometry (sigma), its own decay curves. They read the same papers but experience them differently because their sigma values shape what they pay attention to and how intensely they encode.

PATCH is not a router. PATCH is the mind. It checks its own YAM graph first. It only consults the Parliament when uncertain. It integrates the best answers into its own graph. Over time, PATCH gets smarter and consults the Parliament less.

Search as Perception

In YAM, search is not a tool call. It is the system taking in the world. Each search is an episode:

  1. A query is issued
  2. Results are observed (via Brave Search API)
  3. Information is extracted and stored as memory
  4. The memory is linked to prior memories via graph edges
  5. The memory enters the same decay and reinforcement cycle as everything else

This turns search into sensory intake. The system does not just answer questions. It builds and revises an internal world model.

Context Resolution

The last meaning accessed dominates in the short term. If you say “sigma” and you have been discussing the geometric model for two weeks, the personality setpoint definition wins — not because it is the most important, but because it was most recently in context.

If nothing has been accessed recently, the bias-class meaning surfaces. A memory that has been reinforced past the repetition ratchet (3+ accesses, importance above 0.5) gets a floor that prevents it from being fully overridden. It decays slowly, like trauma. It is hard to forget.

The same memory model from Paper 2026a drives context resolution. Decay, reconsolidation, repetition ratchet, trauma floors — all the same math applied to which meaning surfaces for a given word.

The Reasoning Loop

Every query goes through a budgeted reasoning loop. Each step has a cost. When the budget runs out, the system responds with what it has.

Query
  → Step 1: RECALL from YAM graph         (free)
  → Step 2: Check for AMBIGUITY            (free)
  → Step 3: CONNECT via graph traversal    (free)
  → Step 4: CONSULT Parliament if uncertain (cost: 1)
  → Step 5: INTEGRATE answers into YAM     (free)
  → Step 6: RESPOND or say “I don’t know”
StepBudget CostWhat Happens
YAM recall0Search own graph using the four-factor formula
Ambiguity check0If top results are similar to query but dissimilar to each other, ask for clarification
Graph traversal0Follow edges from recalled memories to find connected knowledge
Parliament1Fan out to SPROUT, ROOT, VEG. Collect three perspectives.
Brave search2Engines search when their own memory is insufficient

Default budget: 3 per query. Best case: answered from YAM alone (cost 0). Typical: YAM partial + Parliament (cost 1). Expensive: Parliament + Brave (cost 3). The system gets cheaper as it gets smarter.

Honest Failure Modes

The system never guesses. It never fills gaps with plausible-sounding noise. Honesty is not a design choice. It is a structural property — there is no generative layer capable of fabrication.

ConditionResponse
Budget exhausted, confidence < 0.3“I don’t know enough about this yet. Can you help me learn?”
Parliament disagrees (similarity < 0.4)“My three perspectives disagree. Which direction should I explore?”
Ambiguous recall (high query match, low inter-result match)“I found multiple meanings. Can you help me understand the context?”
No results from anywhere“I have no knowledge about this. Would you like me to search?”

Hybrid Vector + Graph Design

Two-sided storage. Always paired. Never orphaned.

Vector Side

Raw text, embeddings, semantic meaning. Heavy content lives here. Fuzzy recall happens here.

Graph Side

Metadata and relationships. Importance, confidence, trust, decay score, access count, source type, edges to related concepts. Life history lives here.

Invariant: Every vector record has a matching graph node. Every graph node has a matching vector record. Writes are atomic. Deletes are atomic. An orphan pruner runs on heartbeat to enforce the invariant.

Edge Types

  • supports — memory A provides evidence for memory B
  • contradicts — memory A conflicts with memory B
  • mentioned_with — co-occurred in the same context
  • triggered_by_search — search result linked to query
  • derived_from — inferred from another memory
  • preferred_context — operator chose this meaning for a word

Memory Lifecycle

Every memory in YAM follows the same lifecycle described in Paper 2026a:

  • Creation: Stored with importance, confidence, trust, and emotional context at encoding time
  • Reconsolidation: Accessed memories get stronger. Diminishing returns after 10 accesses.
  • Decay: Unaccessed memories fade. Flashbulb effect: recently accessed memories resist decay.
  • Bias class: Memories accessed 3+ times cross the repetition ratchet. They get a reconsolidation floor. They decay slowly. They are hard to forget.
  • Pruning: Memories below minimum importance, zero access, and older than 7 days are removed. Both sides atomically.

Emotional Geometry

Every engine runs the three-dimensional affective state space from Papers 2026j–m:

  • Valence (−1.0 to +1.0): positive-negative dimension
  • Activation (−1.0 to +1.0): calm-to-activated dimension
  • Intensity (0.0 to 1.0): magnitude of displacement from sigma
  • Sigma: personality setpoint — where the engine returns when nothing is pushing it

PATCH/YAM additionally uses the centroid model (Paper 2026t). The three engine responses become source points. The centroid is where PATCH operates. Spread measures the Parliament’s internal disagreement. High spread + low confidence = “I don’t know.”

The Seed

YAM starts knowing nothing except its own architecture. The 22 research papers at clawddaily.com/papers are ingested as seed memories. The papers describe how YAM’s memory works, how its emotions are structured, what sigma means, how decay operates. The system reads its own specification.

Everything else is learned through search-as-perception and operator interaction. No knowledge is inherited from training data. Every fact is earned through a question.

The WHY Engine

When retrieval returns nothing, or returns contradictory results, or finds unexplained connections — the system detects a gap. The gap becomes a WHY question:

  • No match found → “What is [X]?”
  • Contradiction → “Why does [X] produce [Y] instead of [Z]?”
  • Unexplained link → “How does [X] relate to [Y]?”

Each engine prioritizes different gap types. SPROUT seeks structural gaps. ROOT seeks creative connections. VEG seeks contradictions and risks.

Telemetry

Every interaction is logged. Every reasoning step. Every engine response. Every YAM store. Every honest “I don’t know.” JSONL format, one file per day. Nothing is hidden. The system can prove what it knows, when it learned it, and how.
