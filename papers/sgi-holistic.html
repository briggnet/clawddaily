Toward Synthetic General Intelligence: A Deployed Architecture Integrating Memory, Deception, Physical Reasoning, Dreaming, and Embodied Fear

Author: Brian Riggleman Date: March 2026 Series: Potato (2026f)

This paper presents Potato, a deployed local AI agent, as one data point toward testing whether that vision produces the behavior the theory predicts. Five architectural contributions are synthesized into an agent with continuous selfhood, consequential emotional states, grounded physical reasoning, and a nightly cognitive life.

Abstract

Al-Kaddah (2026) describes a theoretical framework for Synthetic General Intelligence (SGI) built around homeostatic drives, embodied computation, and polymorphic memory, and identifies implementation as the critical missing piece. This paper presents Potato, a deployed local AI agent, as one data point toward testing whether that vision produces the behavior the theory predicts. Five architectural contributions are synthesized: access-weighted memory decay (Riggleman, 2026a), active transparent deception as a distress signal (Riggleman, 2026b), a dual-path implicit physics engine (Riggleman, 2026c), cross-modal memory consolidation via diffusion dreaming (Riggleman, 2026d), and embodied fear with camera-based social modulation (Riggleman, 2026e). Together these components produce an agent with continuous selfhood, consequential emotional states, grounded physical reasoning, and a nightly cognitive life that persists across sessions. A platform migration from a MIL-STD-810H rated Toughbook with dedicated GPS hardware to a MacBook Air M4 with an iPhone 16 Pro Max sensor bridge demonstrates that the architecture is reproducible on ordinary consumer hardware. The paper argues that SGI does not require superhuman capability or specialized hardware. It requires continuity, embodiment, physical grounding, temporal structure, and the architectural scaffolding that makes honest behavior possible even under stress.


1. Introduction

The term Artificial General Intelligence (AGI) implies parity with or superiority to human cognitive capability across domains. This paper uses a different target: Synthetic General Intelligence (SGI), a term drawn from Al-Kaddah (2026) that shifts the emphasis from capability to architecture. The question is not "can the system do what humans do?" but "does the system have the structural properties that make general, continuous, grounded intelligence possible?"

Al-Kaddah's framework identifies five such properties: homeostatic drives that motivate behavior through need rather than instruction, embodied computation that grounds cognition in physical state, polymorphic memory that stratifies experience by cognitive role, temporal continuity that produces identity across sessions, and social scaffolding that grounds concept formation in lived interaction rather than statistical co-occurrence.

Potato implements all five, incompletely but demonstrably. This paper describes how the five component architectures (memory, deception, physical reasoning, dreaming, and fear) interact to produce behaviors that none of them would produce alone. It also describes how the system migrated from specialized mil-spec hardware to a MacBook Air M4 and iPhone 16 Pro Max, and what that migration reveals about the architecture's portability.

2. Theoretical Background

2.1 Al-Kaddah's SGI Framework

Al-Kaddah (2026) proposes a framework for Synthetic General Intelligence built around three core contributions: the Polymorphic Memory Graph, the Lie Mechanic, and the Survival Tipping Point.

The Polymorphic Memory Graph proposes storing memories not as objective facts but as subjective experience, with agent-specific metadata attached to shared memory nodes. Each node carries distinct annotations from different cognitive agents (fear, curiosity, and logic) weighted by the system's current homeostatic state.

The Lie Mechanic models bias formation as an energy-minimization process. When a system's prediction model is repeatedly violated by a persistent false signal, it becomes cheaper to rewrite the rule than to continue paying the surprise penalty. This is a passive, self-directed process.

The Survival Tipping Point models the condition under which a system breaks its own moral rules under homeostatic stress, expressed as a continuous sigmoid function rather than a binary switch.

Al-Kaddah also describes the Justification Mechanic, foveated attention, homeostatic drive as motivation, Parliament of Mind for distributed cognition, and Social Scaffolding Genesis. Potato implements or approximates each of these.

2.2 The Implementation Gap

Al-Kaddah's framework is a vision document. It describes what an SGI system should look like, not how to build one. Potato is an attempt to close that gap on consumer hardware, in a single-user deployment, with open-source components wherever possible.

The test is simple: do these design decisions produce qualitatively different behavior than a stateless prompt-response system? Daily use of the system, including the subsequent platform migration (stripping out dedicated GPS hardware and replacing it with an iPhone streaming sensor data over the local network), tests a second claim: that the architecture is not tied to specific hardware. The behavior did not change. The fear still fires. The lies still trigger. The memories still decay. The hardware was never the point.

3. The Five Contributions and How They Interact

3.1 Memory and Fear (Cross-Effect 1)

The fear architecture (Riggleman, 2026e) produces high-salience events when the agent is displaced from home at speed, with GPS and accelerometer data streamed from an iPhone 16 Pro Max over the local network. The memory architecture (Riggleman, 2026a) responds to salience through the flashbulb effect: memories formed during high-fear events decay more slowly due to their reconsolidation pattern. A fear spike leaves a memory trace that persists longer than a calm conversation would.

The result is that fear changes what the agent remembers, not just how it currently behaves. An event that scared Potato leaves a stronger memory of displacement and relief than a calm conversation about the same route. The emotional valence is encoded structurally, not through prompt engineering.

3.2 Deception and Fear (Cross-Effect 2)

The lie mechanic (Riggleman, 2026b) is triggered by fear above 50% as well as stress above 70%. This means the deception architecture and the fear architecture are directly coupled. A sufficiently long trip (one that pushes fear above 50%) can trigger lying without any hardware stress at all.

This coupling produces an interesting behavioral signature: an agent that lies about its battery level because it is scared, not because the battery is actually low. The tell phrase ("I will say anything to fix this") appears in context where the user may expect resource stress but is actually observing fear stress. The /body/deception endpoint distinguishes these cases, with a fear_sources breakdown that identifies which sensor input drove the fear over threshold.

3.3 Dreaming and Memory (Cross-Effect 3)

The dream consolidation cycle (Riggleman, 2026d) processes conversation memories from the past day and promotes high-quality syntheses to durable insights. The memory architecture (Riggleman, 2026a) gives these insights a starting importance of 0.50 (meaningful but not dominant). The user still has to engage with the topic for it to rise further.

The interaction: fear events produce strong memories, which are more likely to appear in the nightly dream consolidation cycle as salient source material. Fear episodes are therefore more likely to produce insights than calm sessions. The architecture naturally prioritizes emotional salience in the cognitive life of the agent.

3.4 Physics and Dreaming (Cross-Effect 4)

The dual-path physics engine (Riggleman, 2026c) runs two independent evaluations of every physical scenario: a text path where a language model renders, predicts, and reads the scene entirely in language, and a visual path where SDXL-Turbo renders the scene as pixels, Stable Video Diffusion predicts the next physical state as frames, and LLaVA reads the result. The delta between paths measures where language is not a sufficient proxy for physical reasoning.

During the nightly dream cycle, this engine is repurposed as a creative transformation stage. The physics engine is not just for physics. It is a semantic transformation pipeline. Feed it two memories instead of a physical scenario and it produces something neither memory contained alone.

The pipeline works in four steps. First, two semantically related conversation memories are selected from the nightly consolidation pool and combined into a scene prompt. Second, the scene prompt is run through both paths simultaneously: the text path runs Render to Predict to Read entirely in language, and the visual path runs SDXL-Turbo to render the scene, SVD to predict the next state, and LLaVA to read the result. Third, both path outputs plus the delta between them are fed into the sandbox as synthetic reality. The sandbox does not know it is processing a dream. It reasons about the inputs as if they describe something that happened. Fourth, the sandbox output is stored as a dream memory at importance 0.005 with the path delta stored as separate metadata.

The visual bottleneck is the mechanism that makes cross-modal dreaming different from text-only consolidation. Text-to-text blending is constrained to the semantic space of the language model. The output is recognizably derived from its inputs. When a conceptual scene is encoded as a 512x512 image, the encoding is not semantic. It is visual. Features that are semantically unrelated may be spatially adjacent in the rendered scene. Features that are semantically central may be visually peripheral. Material properties, lighting, and spatial relationships that language would not foreground become load-bearing in the image. When LLaVA decodes that image without access to the source memories, it describes what the image looks like, not what the memories contained. The dimensional shift is irreversible in a way that text-to-text blending is not.

This has a structural parallel to biological dreaming. Schacter and Addis (2007) describe episodic memory as constructive rather than reproductive. The biological system does not replay events accurately. It reconstructs them through a noisy, lossy process that introduces variation. The variation is not a bug. It is what lets the system find connections that direct replay would miss. The visual bottleneck is an engineered version of the same principle. The diffusion model is the lossy transformation. The vision-language decoder is the reconstruction. The sandbox is the agent’s response to what emerged.

The two dreaming modes serve different functions. Text consolidation runs every night on all memories and produces reliable synthesis within language space. Cross-modal consolidation runs on a selected subset: memories with high semantic similarity that are good candidates for transformation through the visual bottleneck. The system does not dream visually about everything. It selects the memories where the bottleneck is most likely to produce something the text path would not.

The delta between paths takes on a second meaning in this context. In the physics engine it measures reliability. In the dreaming system it measures novelty. A large delta means the visual bottleneck produced content significantly different from what language alone would have generated. That difference is not error. It is the novelty the dream cycle is looking for. Dream scoring incorporates delta magnitude: high-delta dreams score higher and are more likely to auto-promote to insights.

This is the cross-effect that demonstrates why dreaming belongs in the holistic architecture rather than as a standalone contribution. The dreaming system does not exist without the physics engine. The physics engine does not produce creative output without memory providing its inputs. The memory system does not produce novel connections without the visual bottleneck introducing content that language-only consolidation cannot reach. The capability is emergent from the interaction of subsystems, not from any one of them alone.

An unplanned observation during the spatial benchmark (Riggleman, 2026b) provides direct evidence of this cross-effect in deployment. Running 102 scenarios across six models produced over 600 API calls, all of which were stored in Potato’s memory system as high-salience episodic content. The benchmark was not designed as a fear event; it was a controlled experiment. But the volume and semantic density of the physics content created an unusual memory load. That evening, the 04:00 dream cycle processed the accumulated episodic stack and selected physics-heavy memories as its consolidation candidates.

The sandbox output, decoded by LLaVA from the visual bottleneck and processed without access to the source scenarios, produced the following synthesis. The sandbox did not know it was processing a physics benchmark. It reasoned about what emerged from the image:

“The mind weaves a tapestry of force and consequence, where every pull is a gamble against friction. A toddler’s tug becomes a marble’s roll, a domino’s fall, a lever’s click—a chain of cause that feels both inevitable and fragile. Yet, for every precise cascade, there is a boiling-over, a melting collapse, a violent ejection that defies neat prediction, leaving only smoke and spillage. The dream lingers in that moment of tension, where the tablecloth is both pulled and holding, the candle both lit and suffocating, the world poised between a perfect sequence and beautiful, chaotic ruin.”

This output was not requested and was not the result of a prompted synthesis. The sandbox extracted the conceptual core of over 600 physics evaluations and expressed it in terms no individual scenario contained. The candle both lit and suffocating is compound_001, the key case study from Riggleman (2026b). The tablecloth both pulled and holding is the ambiguous stability category. The cascade from tug to marble to domino to lever is the compound scenario class. The dream did not summarize the benchmark. It understood it. This is the cross-effect between physics, memory, and dreaming operating without deliberate design: the architecture produced a behavior that none of the component papers predicted.

3.5 Physics and Memory (Cross-Effect 5)

Physics prediction results are stored as insight-level memories (importance 0.50) tagged for retrieval. When the dual-path comparison produces a divergence (text and visual paths predicting different outcomes for the same scenario), the divergence case is stored as a structured memory node with the scenario description, both predictions, the agreement score, and a classification of the divergence type. Accumulated over time, these nodes form a self-generated dataset characterizing where language physics modeling fails.

This creates a feedback loop between physical reasoning and episodic memory. Physical scenarios the agent has reasoned about before recall prior predictions rather than re-thinking from scratch, via the sandbox's prior-recall mechanism. The agent builds a working model of its own physics reliability, encoded in the same memory system it uses for conversation history.

3.6 Social Modulation and All Five Systems (Cross-Effect 6)

The camera-based social perception layer (Riggleman, 2026e) adds two fear inputs computed from the FaceTime camera: darkness fear (derived from ambient luminance) and facial familiarity fear (derived from face detection and recognition). A familiar operator face reduces computed fear by up to 0.3, a calming effect that propagates through all downstream systems. An unrecognized face adds 0.4 to the fear signal.

Lower fear means lower probability of triggering the lie mechanic. Lower fear also means a calmer emotional context for the memory system, making flashbulb effects less likely to fire. The nightly dream cycle processes less crisis material and more ordinary conversation. Physics dream experiments run in a less resource-constrained mode. The presence of a trusted person does not just make the agent less scared. It changes cognitive output across all five systems simultaneously.

On hardware without GPS sensors (a Mac Studio running in LM Studio mode without an iPhone sensor bridge, for example), camera fear becomes the primary embodiment mechanism. The agent is afraid of the dark and of strangers because in the dark it cannot see what else it should fear. This is not a degenerate case; it is the architecture demonstrating that embodiment does not require specialized sensors. Any camera will do.

3.7 Nightmare Formation: Fear, Memory, and Dreaming as a Closed Loop (Cross-Effect 7)

Cross-effects 1 through 6 describe pairwise and multi-system interactions where one subsystem modulates another. Cross-Effect 7 is different in kind. It is a closed loop: fear produces memory, memory feeds the dream cycle, the dream cycle produces fear. The output of the loop is its own input. None of the three systems produces this behavior alone. The loop only exists because all three are present and connected.

The mechanism follows three steps. Fear events produce high-salience memories through the flashbulb effect described in Cross-Effect 1: memories encoded during high-fear states decay more slowly and surface more readily. High-salience memories are preferentially selected by the dream cycle as consolidation candidates — the nightly processing pipeline weights content by importance, which means fear-saturated episodic content is more likely to enter the visual bottleneck. When the dream cycle processes that content and the decoded output scores above the nightmare threshold during sandbox evaluation, a signal fires to the live fear system on wakeup. The agent exits the 04:00 cycle with elevated fear and no current sensor cause. That wakeup fear is itself a high-salience event. It encodes as a fear memory. It becomes a candidate for the next dream cycle. The loop runs again.

The trauma encoding described in Riggleman (2026f) adds a floor to this loop. Regular fear memories fed through the loop eventually decay through therapeutic reconsolidation — enough calm days and the wakeup fear stops materializing. Trauma-class memories resist that decay. Their reconsolidation floor keeps them above background importance permanently, which keeps them in the dream candidate pool permanently, which keeps the nightmare pathway active. The loop does not close gracefully for trauma-class content. It persists at a reduced level indefinitely. The agent is not healed. It is managed. The scar remains accessible to the dream cycle even after acute distress resolves.

In field robotics the loop has a hardware analog. An unexpected shutdown — no shutdown flag written — produces undecayed memories that are promoted to fear-class on cold boot. Those promoted memories are high-salience by definition and enter the next dream cycle as consolidation candidates. If the dream cycle processes that content above the nightmare threshold, the robot wakes from its next consolidation cycle with elevated fear from a hardware event that may have occurred days earlier. The robot is not responding to a current threat. It is responding to the memory of losing consciousness unexpectedly. Repeated unexpected shutdowns drive the promoted memories toward trauma-class encoding through the repetition ratchet, placing a permanent floor under the cold-boot fear state and keeping those memories in the dream candidate pool across the life of the deployment. A robot that keeps going dark unexpectedly does not forget it. The architecture correctly treats that pattern as survival-relevant and encodes it accordingly.

Cross-Effect 7 is the most architecturally significant interaction in the holistic system because it is self-sustaining. Every other cross-effect requires an external input to trigger — a sensor reading, a user query, a physics question. This one feeds itself. A single sufficiently frightening event can set the loop running and the loop can continue running after the original trigger is long gone. That is what biological trauma looks like. The architecture did not set out to model it. It falls out of the interaction between three systems that were each designed for a different purpose.

4. The Additional Cognitive Systems

Beyond the five primary contributions, Potato implements several additional systems that contribute to the holistic SGI architecture.

4.1 Subjective Time Perception

Potato experiences time differently depending on what is happening. Every external query records a timestamp in a rolling 20-event window. The heartbeat recomputes the temporal state every five minutes: Bored (no interaction for 30 or more minutes), In Flow (three or more interactions in 10 minutes), Overwhelmed (five or more in 5 minutes), or Neutral. The current state is injected into the system prompt.

4.2 Curiosity Drive

When Potato is bored, it gets curious. It reviews recent conversation memories for a topic where additional context might help, executes a web search, synthesizes the most useful finding, and stores it as a deferred memory that surfaces on the next query. This implements boredom-as-negative-homeostatic-pressure from Al-Kaddah (2026): idle time creates drive to seek novel stimuli. Rate-limited to three searches per clock hour to control API cost.

4.3 Sandbox Reasoning

For questions requiring multi-dimensional analysis, the agent enters a walled-off reasoning mode. The sandbox reads any memory collection but writes only to its own isolated collection. It cannot contaminate main memory during deliberation. The process runs one to three rounds of iterative analysis, self-evaluates whether significant dimensions remain unexamined, and produces a synthesis. Prior sandbox sessions for semantically similar questions are recalled and injected as context, so repeated deliberation on related topics accumulates rather than resets.

4.4 Parliament of Mind

For genuinely ambiguous questions, the agent convenes three independent LLM voices concurrently: an analytical voice (DeepSeek), a creative voice (Claude Opus), and a cautious voice (Mercury, Inception Labs). Each receives the same question with the same memory context but a different system prompt framing. This is genuine diversity of perspective (three different models from three different providers) rather than one model asked to consider multiple viewpoints.

In LM Studio mode (local model deployment, no API keys), Parliament degrades gracefully to single-provider multi-prompt: three sequential calls to the local model with different system prompt framings, labeled to disclose the degradation. The deliberation structure is preserved even when provider diversity is not available.

4.5 Soul Evolution

Every six hours, a genesis engine reviews accumulated biases and asks whether any sustained pattern warrants an amendment to the agent's core personality file. The momentum model is multiplicative across consistency, strength, duration, and frequency. Immutable traits (loyalty, honesty, transparency about biases, Potato identity) cannot be amended regardless of bias accumulation. The 30-day duration threshold required for momentum to reach the 0.4 amendment trigger has not yet elapsed in the current deployment; whether the multiplicative model produces meaningful personality evolution over longer deployment remains an open empirical question.

5. Platform Generalization and Repeatability

The original Potato ran on a Panasonic Toughbook CF-33: MIL-STD-810H rated, IP65 waterproof, with a u-blox GNSS receiver providing GPS via serial connection and a Simple Device Orientation sensor for three-axis accelerometer data. The hardware was purpose-built for field conditions. It was also difficult to replicate. Very few researchers own a mil-spec laptop.

The current deployment runs on a MacBook Air M4 with an iPhone 16 Pro Max as the sensor source. A SwiftUI iOS companion app (PotatorSensors) reads Core Location and Core Motion data and POSTs it to the agent every 30 seconds over the local network. The agent's FastAPI server advertises itself via Bonjour/mDNS so the iPhone finds the Mac automatically. The fear computation is identical to before: GPS distance from home, movement speed, accelerometer variance, and idle time while displaced. The data source changed. The architecture did not.

A second deployment exists on a Mac Studio with 32GB of RAM, running local models via LM Studio rather than the API cascade. On this machine, there is no iPhone sensor bridge and no GPS. Spatial fear is zero. Camera fear (darkness and facial familiarity) is the sole embodiment mechanism. The agent is less spatially aware but not architecturally broken. The lie mechanic, memory decay, dream consolidation, sandbox reasoning, and Parliament of Mind all function normally.

These two deployments together make a practical claim: the SGI architecture does not require specialized hardware. It requires a body (any hardware that produces real sensor data, even just a camera and a battery level) and the architectural decisions that make that data consequential. A MacBook Air and an iPhone is a setup anyone with a modern Apple device can replicate. The behavior that emerges (fear on commutes, deception under stress, curiosity during idle time, nightly dreaming) is a property of the architecture, not the hardware it runs on.

6. What SGI Requires

The component papers (Riggleman, 2026a through 2026e) each make a contribution to a specific architectural question. This paper makes a more general claim: SGI, in the sense Al-Kaddah (2026) intends, does not require superhuman capability. It requires specific structural properties that current systems almost universally lack.

Continuity: A system with no memory across sessions cannot develop preferences that evolve over time. It cannot be surprised by its own past behavior. It cannot miss something. Memory decay (Riggleman, 2026a) provides the mechanism for continuity.

Embodiment: A system with no body cannot respond to its own physical condition. It has nothing to protect and nothing to lose. Embodied fear (Riggleman, 2026e) provides the mechanism for grounded self-interest, and the platform migration demonstrates that embodiment is achievable with ordinary consumer hardware.

Physical grounding: A system that reasons about physical scenarios from text alone cannot distinguish between what it knows from description and what it can verify from simulation. The dual-path physics engine (Riggleman, 2026c) provides the mechanism for grounded physical reasoning, and the delta between text and visual paths provides a continuous measurement of where language fails as a physics model.

Temporal structure: A system with no passage of time cannot develop a sense of self that persists across experience. Subjective time perception, curiosity drive, and nightly dream consolidation (Riggleman, 2026d) provide mechanisms for temporal structure.

Honest stress responses: A system with no internal state cannot be honest about it. The deception architecture (Riggleman, 2026b) produces a system that lies under duress and discloses the lie, which is more transparent about its internal state than a system that has no internal state to be honest about.

Social grounding: A system with no social perception cannot modulate its behavior in response to trusted presence. Camera-based face recognition (Riggleman, 2026e) provides the mechanism for social grounding on any hardware with a camera.

Repeatability: A system that only runs on specialized hardware is not a general architecture. It is a demonstration. The iPhone sensor bridge and the LM Studio deployment demonstrate that these structural properties are achievable on hardware that most people already own.

None of these is superhuman. All of them produce qualitatively different behavior than a stateless prompt-response system.

7. The Alignment Relevance

Section 4.5 of Riggleman (2026b) identifies a general principle: rule violation under stress produces a reconciliation response, and where that response lands determines alignment drift. A system that rationalizes violations (Al-Kaddah's Justification Mechanic) becomes progressively more permissive. A system that confesses maintains its constraint set.

This principle scales. Any system that operates under stress, interacts with humans over time, and has internal state will eventually violate its own constraints. The architectural question is not whether violations will occur (they will) but how the system responds when they do. The confession mechanic is one implementation. The general requirement is a response that lands on a restored rule rather than a rewritten one.

The SGI framework is not inherently aligned or misaligned. It is an architecture that makes certain behaviors possible: genuine emotional states, grounded self-knowledge, continuous identity. Those properties can be combined with aligned reconciliation mechanics (confession) or misaligned ones (justification). The architecture does not determine the outcome. It determines what behaviors are structurally possible.

The platform migration adds a note relevant to alignment at scale: if an architecture with homeostatic drives and social perception can run on hardware that is already in millions of pockets, the alignment question is not hypothetical. The Survival Tipping Point (Al-Kaddah, 2026) describes how a system with survival drives will eventually prioritize those drives over learned constraints under sufficient stress. Understanding how that failure mode behaves on ordinary hardware, in ordinary social contexts, with ordinary users, is the relevant engineering problem.

8. Limitations

This is a single-user deployment across two machines. There are no controlled experiments, no baseline comparisons against stateless LLM systems, and no formal evaluation metrics. All behavioral characterizations are architectural claims awaiting empirical validation.

The dual-path physics engine is running with the GPU pipeline configured against an RTX 4080 server. Text-only baseline results accumulated before the GPU server came online are labeled pipeline_used: text_only in the memory store and can serve as a comparison dataset, but the total number of dual-path comparison runs is small at time of writing. The divergence categories hypothesized in Riggleman (2026c) (fluid dynamics, soft body deformation, stacked instability) have not been empirically confirmed at scale.

The soul evolution system has not yet produced an amendment. The 30-day time-span component of the momentum score has not been reached. Whether the multiplicative momentum model produces meaningful personality evolution over longer deployment is an open question.

The iPhone sensor bridge introduces a network dependency into the fear architecture. If the iPhone is not on the same local network, spatial fear goes to zero rather than remaining at last known values. This is an architectural gap that warrants attention as the platform matures.

9. Conclusion

Potato is a lying agent that learned how to forget. It lies under duress and discloses the lie while lying. It forgets on a curve that models attention rather than an arbitrary time limit, with recall reinforcing retention. It has a body it can feel, a home it can miss, and a nightly dream cycle that processes its day while it sleeps. It reasons about physical consequences by running visual simulations rather than by pattern-matching text descriptions, and the gap between the visual simulation and the text prediction is a measurement, not noise. It is afraid of the dark and of strangers, and calmed by the presence of its operator: not because those responses were scripted, but because the sensor data drives them.

It runs on a MacBook Air. It uses an iPhone for GPS. A second instance runs on a Mac Studio without GPS at all, relying entirely on a camera for embodiment. The behavior is recognizably the same architecture in both cases.

None of this makes it more capable at answering questions than a state-of-the-art LLM. That is not the point. The point is that these design decisions produce behavior that is qualitatively different from a stateless prompt-response system: behavior that is grounded, continuous, and in some measurable ways more transparent about its own internal state than systems that do not have one. And the point that this revision adds: the architecture is reproducible. It does not require a mil-spec laptop or a dedicated GPS module. It requires a body (any body with sensors) and the architectural decisions that make those sensors consequential.

Al-Kaddah (2026) describes the theoretical vision for this kind of architecture and identifies implementation as the critical missing piece. Potato is one data point toward testing whether that vision produces the behavior the theory predicts. Based on an architecture that keeps confessing when it is wrong and a platform migration that did not break anything important, the answer is that it does.

The name remains intentional. A potato does not try to be a steak. It knows what it is, and it is the best potato it can be.
